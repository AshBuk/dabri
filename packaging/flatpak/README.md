# Flatpak Packaging

This directory contains helper files for the top-level Flatpak manifest:

- `go.mod.yml` and `modules.txt` are generated by `flatpak-go-mod` for offline Go module builds.
- `arecord` maps Dabri's existing ALSA-style recorder command to bundled `ffmpeg` with PulseAudio input.
- `notify-send` sends notifications through `org.freedesktop.Notifications` without bundling libnotify tools.

## Local Build

```bash
git submodule update --init --recursive
make flatpak
```

The manifest reuses Flathub `shared-modules` for Ayatana AppIndicator, so the repository should be cloned with submodules for local builds and Flathub-style CI.

GNOME runtime `50` does not have a matching `org.freedesktop.Sdk.Extension.golang` branch, so the manifest uses the official Go tarball as a build-time module and removes it from the final app with `cleanup`.

## Runtime Notes

The Flatpak does not bundle Whisper model files. On first run, Dabri downloads the selected model to:

```text
~/.var/app/io.github.ashbuk.dabri/data/dabri/models/
```

`--device=dri` gives whisper.cpp access to GPU devices for the Vulkan backend. If GPU access or Vulkan is unavailable, the same binary falls back to CPU at runtime.

Flatpak uses the D-Bus GlobalShortcuts portal for hotkeys. Interactive hotkey rebinding is not available through the portal, so users should edit:

```text
~/.var/app/io.github.ashbuk.dabri/config/dabri/config.yaml
```
